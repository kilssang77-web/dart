# UI 디자인 가이드

---

## 디자인 원칙

1. **도구처럼 보여야 한다** — 입찰 담당자가 매일 쓰는 업무 도구다. 마케팅 페이지처럼 화려하게 만들지 않는다.
2. **숫자가 주인공이다** — 낙찰 확률, 금액, 비율이 최우선 요소다. 장식은 숫자를 돋보이게 할 때만 허용한다.
3. **상태를 명확히 드러낸다** — 로딩(시뮬레이션 진행 중), 에러(API 실패), 빈 상태(공고 없음)를 항상 처리한다.
4. **밀도 우선** — 한 화면에서 많은 입찰 정보를 소화해야 한다. 여백을 아끼되 과밀하지 않게 한다.

---

## AI 슬롭 안티패턴 — 하지 마라

| 금지 패턴 | 이유 |
|-----------|------|
| `backdrop-filter: blur()` | glass morphism은 AI 템플릿 1번 징후 |
| gradient-text (배경 그라데이션 텍스트) | 업무 도구에 어울리지 않음 |
| 어두운 그라데이션 Hero 섹션 | 랜딩 페이지가 아님 |
| 카드 hover 시 float·scale 효과 남용 | 테이블형 데이터에 불필요한 시각적 노이즈 |
| 색상 강조 3가지 이상 동시 사용 | 단일 강조색 원칙 위반 |

## 테마

- **모드**: 라이트 모드 단일
- **색상 토큰**: Tailwind CSS config에서 관리

```
primary:  slate-800  (주 텍스트, 사이드바 배경)
accent:   blue-600   (버튼, 링크, 강조 수치)
surface:  white / slate-50  (카드 배경)
border:   slate-200
muted:    slate-400  (보조 텍스트, 플레이스홀더)
danger:   red-500    (에러, 낙찰 실패)
success:  green-600  (낙찰 성공, 긍정 지표)
```

---

## 컴포넌트 시스템

- **라이브러리**: shadcn/ui (Radix UI 프리미티브 + Tailwind CSS)
- **설치 방식**: `npx shadcn@latest add <component>` (소스 복사)
- **CSS-in-JS**: 사용 안 함 — Tailwind 유틸리티 클래스만 사용

### 컴포넌트 사용 우선순위

1. shadcn/ui 컴포넌트 그대로 사용
2. `components/ui/`에 shadcn 기반 래퍼 생성
3. 완전 커스텀 (최후 수단)

## 레이아웃

```
┌─────────────────────────────────────────────────┐
│  사이드바 (240px)  │  메인 콘텐츠 영역            │
│  - 로고            │  - 페이지 헤더               │
│  - 네비게이션      │  - 필터 / 검색 바            │
│  - 사용자 정보     │  - 테이블 / 카드 목록        │
│                   │  - 페이지네이션              │
└─────────────────────────────────────────────────┘
```

- 사이드바 너비: 240px (고정)
- 최소 지원 화면 너비: 1024px (데스크탑 우선)
- 콘텐츠 최대 너비: 1280px (max-w-screen-xl)

---

## 타이포그래피

| 용도 | 클래스 |
|------|--------|
| 페이지 제목 | `text-2xl font-semibold` |
| 섹션 제목 | `text-lg font-medium` |
| 본문 | `text-sm` (14px) |
| 보조 텍스트 | `text-xs text-slate-500` |
| 금액/확률 강조 | `text-xl font-bold text-blue-600` |

- 폰트: 시스템 폰트 스택 (`font-sans`)
- 한국어 환경 기준: Noto Sans KR 폴백 없이 시스템 폰트 우선

---

## 폼팩터 및 반응형

- **기준**: 데스크탑 우선 (1024px+)
- **모바일**: 이번 prototype 범위 밖 — 반응형 구현 생략
- 브레이크포인트: Tailwind 기본값 사용 (sm:640, md:768, lg:1024, xl:1280)

## 상태 처리 패턴

모든 비동기 작업은 로딩·에러·빈 상태를 반드시 처리한다.

### 로딩
- 테이블 스켈레톤 or Spinner (Radix `@radix-ui/react-scroll-area` 내 스크롤 중 유지)
- Monte Carlo 시뮬레이션 진행 중: `"분석 중... (30,000회 시뮬레이션)"` 메시지 표시

### 에러
- Toast (shadcn `sonner` 또는 `toast`) — 일시적 오류
- Inline 에러 메시지 — 폼 검증 오류

### 빈 상태
- 아이콘 + 안내 문구 (예: "조건에 맞는 공고가 없습니다")
- 검색 조건 초기화 버튼 제공
