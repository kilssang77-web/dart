## 개요

승률 향상을 목표로 한 10개 step 연속 구현. inpo21c 실측 데이터를 ML 파이프라인 전반에 통합하고, 사용자가 입찰 전략을 더 정밀하게 수립할 수 있는 신규 UI 5종을 추가했다.

- BidDetailPage '공동도급 시뮬레이터' 버튼 추가

### Step 7 — inpo21c 실측 사정율로 ML 예정가격 예측 정확도 향상

**Backend**
- `SRATE_FEATURE_COLS` inpo21c 3개 특성 추가: inpo21c_srate_mean / std / n
- `train_srate_model()` inpo21c LEFT JOIN 쿼리 추가
- `simulate_yejung()` spread 클램프 수정: ±2% → ±2.8% (inpo21c 실측 p02~p98 기반)
- `predict_srate()` Bayesian 블렌딩: n≥3 시 w=min(0.85, n/(n+10)) 적용
- `get_inpo21c_pattern_direct()` 신규 (agency_id → yega stats 직접 조회)
- ML 재훈련 완료 (8,801건)

**검증:** 대전광역시 사정율 0.874 → 0.9854 (실측 0.9983, 오차 -1.3%p)

### Step 8 — 사정율 출처·신뢰도 표시 (bid-range API 확장)

**Backend**

- `/recommend/bid-range` 라우터에서 소스 정보 pass-through

**Frontend**

- 신뢰도 가로 진행 바 추가

### Step 9 — 복수예가 위치 가중 추첨 시뮬레이션

**Backend**
- `simulate_yejung()` — `pos_weights` 파라미터 추가 (15개 위치별 추첨 가중치)
- Gumbel-max trick으로 벡터화된 가중 비복원 추첨 구현 (n_sim=30,000)
- `prism.py`, `services.py` — 에이전시별 실측 pos_weights 자동 적용

**수학 기법:** `np.argsort(log(w) + Gumbel(0,1), axis=1)[:, -4:]`
