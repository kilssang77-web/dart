## 주요 변경사항

### Step 1 — Notification 모델 + 알림 생성 서비스
- `models.py`: `notifications` 테이블 추가 (`ntype`, `title`, `body`, `link`, `is_read`, `user_id`)
- `schemas.py`: `NotificationOut`, `NotificationListResponse` 추가
- `services.py`: `NotificationService` 7개 메서드 구현
  - `create()`, `list_for_user()`, `mark_read()`, `mark_all_read()`, `unread_count()`
  - `create_keyword_match()`: WatchKeyword 활성 사용자에게 개인 알림 생성
  - `create_srate_spike()`: 전체 공지 알림 생성

### Step 2 — 키워드 매칭 공고 알림 자동 생성
- `collector/service.py`: `_check_keyword_match()` 헬퍼 추가
- 신규 공고(`is_new=True`) 수집 시 WatchKeyword 매칭 → 알림 자동 생성
- 오류 발생 시 로그만 기록, 수집 작업 중단 없음

### Step 3 — 사정율 급변 알림 자동 생성
- `collector/scheduler.py`: `run_srate_spike_check_job()` 추가
  - 매일 07:00 KST 실행 (`srate_spike_check_daily`)
  - `SrateTrendService.get_top_trends()` 호출 — ±2%p 이상 급변 발주처 탐지
  - `NotificationService.create_srate_spike()` 호출 → 전체 공지

### Step 4 — Notification API + 헤더 알림 뱃지 UI
- 백엔드 API 4개:
  - `GET /api/v1/notifications` — 목록 조회 (unread_only, limit)
  - `GET /api/v1/notifications/unread-count` — 미읽음 개수
  - `POST /api/v1/notifications/{id}/read` — 개별 읽음 처리
  - `POST /api/v1/notifications/read-all` — 일괄 읽음 처리
- 프론트엔드:
  - `types/index.ts`: `Notification`, `NotificationListResponse` 타입 추가
  - `api/index.ts`: `notificationsApi` 추가
  - `AppLayout.tsx`: 사이드바 벨(🔔) 아이콘 + 미읽음 뱃지 (60초 폴링)

### Step 5 — 알림 목록 페이지 + 읽음 처리
- `NotificationsPage.tsx` 신규
  - 알림 목록 (최대 50개), 미읽음 파란 점 강조
  - 개별/일괄 읽음 처리
  - ntype 뱃지 색상 (키워드=파랑, 사정율=주황, 시스템=회색)
  - link 있는 알림: 클릭 시 해당 페이지로 이동
- `App.tsx`: `/notifications` 라우트 등록
