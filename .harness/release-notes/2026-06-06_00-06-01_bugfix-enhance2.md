## 수정 내역

### 버그 수정

- **OpportunityScoreService timezone 오류** (`services.py`): `datetime.now()` → `datetime.now(timezone.utc)` 변경. PostgreSQL `TIMESTAMPTZ` 컬럼과 naive datetime 비교 시 `TypeError: can't compare offset-naive and offset-aware datetimes` 발생하여 `GET /bids/recommended` 500 오류 → 정상화
- **JointQualService subquery 언패킹 오류** (`services.py`): `db.query(Competitor, stats_subq)` 형태로 조회 시 subquery 컬럼이 개별 확장되어 `ValueError: too many values to unpack (expected 2)` 발생. `db.query(Competitor, stats_subq.c.total_bids, stats_subq.c.win_count, stats_subq.c.avg_rate)` 로 컬럼 명시 분리하고 `competitor, _tb, _wc, _ar` 4원 언패킹으로 수정 → `GET /bids/{id}/joint-partners` 정상화
- **DashboardPage TypeScript 빌드 오류** (`DashboardPage.tsx`): Lucide `<Info>` 아이콘에 `title` prop 전달 시 TypeScript 오류 발생. `<span title={...}><Info /></span>` 래퍼로 교체하여 빌드 통과

### 데이터 보강

- **assessment_rate_stats 월별 집계 추가**: 기존에 연간(annual) 집계만 있어 `GET /stats/top-srate-trends` 가 0건 반환. `bid_results` 기반 SQL UPSERT로 2025-05~2026-06 월별 집계 2,366건 삽입 (agency 2,356건 + global 10건, 샘플 3건 이상 기관만)

### 서비스 로직 개선

- **SrateTrendService 조회 윈도우 확장** (`services.py`): `get_top_trends` 및 `_from_bid_results` SQL의 조회 기간 6개월 → 12개월 확장. 실데이터가 2025-05~11에 집중되어 있어 6개월 윈도우로는 탐색 불가했음
- **`_build_result` fallback 로직 추가** (`services.py`): recent 기간(최근 6개월) 데이터가 없을 때 "최근 데이터 없음"으로 early-return하던 것을 → 보유 데이터를 시간순 정렬 후 후반절/전반절로 자동 분할하여 트렌드 계산하는 fallback으로 교체
