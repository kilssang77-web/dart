## 1. 시스템 개요

### 1.1 시스템 목적

G2B 입찰 분석 시스템은 **조달청 나라장터(G2B)** 건설공사 입찰 데이터를 자동 수집·분석하여, 입찰 참여 의사결정을 지원하는 전문 분석 플랫폼입니다.

핵심 기능:
- 나라장터 최신 공고 자동 수집 (일 2회) + 낙찰 결과 자동 연계
- 낙찰 이력 기반 AI 투찰률 추천 (로컬 ML, 외부 AI 미사용)
- A값·낙찰하한가 자동 계산 + 프리즘 2.0 구간별 낙찰확률 히트맵
- 사정율 트렌드 알림 (발주처×공종 최근 3개월)
- 공고 자동 추천 TOP5 (점수 기반 — 경쟁·발주처 트랙·트렌드·금액)
- 경쟁사 행동 패턴 분석 + 투찰구간 실시간 모니터링
- 낙찰 후 역산 분석 (내 투찰률 vs 낙찰자 격차·편향 보정)
- 예가 번호 패턴 분석 (발주처 특화)
- 공동도급 적격심사 AI 매칭 + 전략 레포트 PDF 출력
- inpo21c 전 참여자 데이터 수집 (31,800건+)
- ML 예정가격 예측 신뢰도 뱃지 (inpo21c 실측 N건 기반 Bayesian 사정율 반영)
- 발주처 예가 위치 패턴 분석 탭 (15개 번호별 추첨 가중치 — AgencyDetailPage)
- 복수예가 위치 가중 시뮬레이션 (발주처별 실측 가중치 반영, 예정가격 분포 정확도 향상)
- 경쟁사 행동 예측 ML (특정 공고 참여 확률 + 예상 투찰구간 예측)
- 자사 승률 패턴 진단 (월별·구간별·공종별 승률 분석)
- 공동도급 적격심사 시뮬레이터 v2 (귀사+경쟁파트너 점수 비교 + 지분율 시뮬레이션)

### 1.2 기술 스택

| 구성요소 | 기술 |
|---------|-----|
| 데이터 수집 | Python 3.11 + requests (G2B Open API) |
| 백엔드 | FastAPI + SQLAlchemy + PostgreSQL 16 |
| 프론트엔드 | React 18 + TypeScript + TailwindCSS |
| ML 모델 | XGBoost Quantile Regression + LightGBM (로컬 실행) |
| 캐시 | Redis 7 |

### 1.3 데이터 수집 일정

| 수집 시각 | 내용 |
|---------|-----|
| 매일 06:00 | 신규 공고 수집 (G2B OpenAPI) |
| 매일 18:00 | 낙찰 결과 수집 + 내 입찰 이력 자동 연계 |
| 매일 09:00 | inpo21c 낙찰 사전정보 수집 (전일 개찰 완료 건) |
| 매주 월요일 02:00 | inpo21c 전 참여자 데이터 수집 (최대 10페이지) |

> **참고:** 데이터는 조달청 나라장터 OpenAPI (`BidPublicInfoService`) 및 inpo21c 기준이며, API 제공 시각에 따라 실제 반영까지 수 시간 지연이 발생할 수 있습니다.

## 2. 로그인 및 접근 권한

### 2.1 로그인

1. 브라우저에서 `http://localhost:3001` 접속
2. 이메일·비밀번호 입력 후 **로그인** 클릭
3. 로그인 성공 시 대시보드로 자동 이동

> 최초 관리자 계정은 시스템 초기 설정 시 생성됩니다. 계정이 없는 경우 관리자에게 문의하세요.

### 2.2 사용자 역할

| 역할 | 권한 |
|------|-----|
| **admin (관리자)** | 전체 기능 + 사용자 관리 + 시스템 현황 |
| **analyst (분석가)** | 입찰·경쟁사·통계·AI 추천·키워드 관리 |
| **viewer (뷰어)** | 읽기 전용 (입찰·경쟁사·통계 조회) |

### 2.3 세션 관리

- 로그인 상태는 브라우저 로컬 스토리지에 JWT 토큰으로 유지됩니다.
- 토큰 만료 시 자동으로 로그인 페이지로 이동합니다.
- 다른 기기에서 로그인해도 기존 세션은 유지됩니다.

## 3. 대시보드

**경로:** `/dashboard`  
**접근 권한:** 전 역할

### 3.1 화면 구성

대시보드는 시스템 전체 현황을 한눈에 파악하는 요약 화면입니다.

#### 핵심 지표 카드 (4개)

| 지표 | 설명 | 산출 기준 |
|-----|------|---------|
| **전체 입찰 (24개월)** | DB에 저장된 최근 24개월 입찰 공고 총 건수 | `bids` 테이블 notice_date 기준 |
| **등록 경쟁사** | 시스템에 등록된 경쟁사 업체 수 | `competitors` 테이블 전체 |
| **평균 낙찰률** | 개찰 완료된 공고의 낙찰 투찰률 평균 | 낙찰자 bid_rate 평균 (소수점 → %) |
| **평균 경쟁강도** | 공고당 평균 참여업체 수 | bid_results 기준 공고별 참여사 수 평균 |

> **낙찰률 정의:** 낙찰 투찰금액 ÷ 기초금액 × 100 (%). 통상 87~92% 범위에서 형성됩니다.

#### 월별 추이 차트

- **막대(파란색):** 월별 입찰 건수 (좌축)
- **꺾은선(진파란):** 월별 평균 낙찰률 % (우축, 87~92% 범위)
- 24개월 데이터 표시

#### 발주기관별 입찰 건수 TOP 10

- 최근 12개월 기준
- 수평 막대 차트로 발주기관별 입찰 건수 비교
- 입찰 참여 빈도가 높은 기관 파악에 활용

#### 사정율 트렌드 알림 카드 (신규)

최근 3개월 사이 사정율이 급변한 발주처×공종 조합 상위 3건을 표시합니다.

| 뱃지 | 의미 |
|-----|------|
| ↑ 빨강 | 사정율 급등 — 예정가격이 높아지는 추세 |
| ↓ 파랑 | 사정율 급락 — 예정가격이 낮아지는 추세 |
| → 회색 | 변화 없음 |

> **활용:** 사정율 상승 추세 기관에 입찰 시 A값도 함께 높아질 가능성이 있어 낙찰하한가 계산에 주의합니다.

#### 이번 주 추천 공고 TOP5 (신규)

7일 이내 마감 예정 공고 중 시스템이 점수화한 상위 5건을 표시합니다.

| 표시 항목 | 설명 |
|---------|------|
| **등급 뱃지** | S(90+) / A(80+) / B(70+) / C — 종합 점수 등급 |
| **점수 바** | 경쟁(40) + 발주처 트랙(30) + 트렌드(15) + 금액(15) |
| **개찰일·기초금액** | 마감 임박도와 규모 확인 |
| **상세 링크** | 클릭 시 입찰 상세 페이지로 이동 |

### 3.2 활용 방법

1. **시장 동향 파악:** 월별 추이 차트에서 입찰 건수 증감과 낙찰률 변화를 확인하여 시장 경쟁 강도를 판단합니다.
2. **주력 기관 확인:** TOP 10 차트에서 발주 빈도가 높은 기관을 파악해 영업 우선순위를 설정합니다.
3. **트렌드 알림 활용:** 사정율 급변 알림에서 관심 기관의 최근 동향을 파악합니다.
4. **추천 공고 검토:** 이번 주 TOP5 공고를 매일 확인해 놓치는 공고 없이 참여 여부를 판단합니다.

## 4. 입찰현황

**경로:** `/bids`  
**접근 권한:** 전 역할

### 4.1 화면 구성

조달청 나라장터에서 수집한 건설공사 입찰 공고 목록을 표시합니다.

#### 필터 및 검색

| 필터 | 설명 |
|-----|------|
| **키워드 검색** | 공고명·발주기관 텍스트 검색 (Enter 또는 돋보기 클릭) |
| **상태 필터** | 전체 / 공고중(open) / 개찰완료(closed) |
| **정렬** | 공고일순 (최신 공고 우선) / 개찰일순 (임박 개찰 우선) |

#### 목록 컬럼

| 컬럼 | 설명 |
|-----|------|
| **공고명** | 조달청 입찰 공고명 (G2B 수집: 파란 배지, 데모데이터: 회색 배지) |
| **발주기관** | 발주 기관명 |
| **지역/공종** | 공사 지역·업종 분류 |
| **기초금액** | 예정가격 산정 기준이 되는 금액 (억원 단위) |
| **공고일** | 조달청 공고 등록일 |
| **상태** | 공고중(입찰 진행) / 개찰완료(결과 확정) |

#### 페이지네이션

- 페이지당 20건 표시
- 이전/다음 버튼으로 페이지 이동

### 4.2 활용 방법

1. **최신 공고 확인:** 기본 정렬이 공고일 내림차순이므로 상단에 최신 공고가 표시됩니다.
2. **임박 개찰 파악:** 정렬을 '개찰일순'으로 변경하면 곧 개찰될 공고를 우선 확인할 수 있습니다.
3. **관심 기관 필터:** 키워드에 기관명을 입력해 특정 발주처 공고만 조회합니다.
4. **상세 분석:** 목록의 행을 클릭하면 [입찰 상세](#5-입찰-상세) 화면으로 이동합니다.

### 4.3 데이터 수집 기준

- **수집 대상:** 조달청 OpenAPI `getBidPblancListInfoCnstwk` (공사 입찰공고 목록)
- **수집 범위:** `inqryDiv=1` (전체 입찰방식)
- **수집 주기:** 09:00, 15:00 자동 수집 (당일 공고 기준)
- **저장 조건:** 동일 `bidNtceNo`(공고번호)가 이미 존재하면 업데이트, 없으면 신규 저장
- **최대 소급 범위:** 시스템 최초 구동 시 최근 180일 소급 수집

## 5. 입찰 상세

**경로:** `/bids/:id`  
**접근 권한:** 전 역할

### 5.1 화면 구성

#### 기본 정보 영역

| 필드 | 설명 | 비고 |
|-----|------|------|
| **발주기관** | 공고를 낸 기관명 | |
| **지역** | 공사 현장 지역 | |
| **공종** | 업종 분류 (예: 토목공사, 건축공사) | |
| **기초금액** | 예정가격 산정 기준금액 | 억원 단위 |
| **예정가격** | 예정가격 (기초금액 × 복수예비가격 적용) | 있을 경우 표시 |
| **낙찰하한율** | 이 비율 미만 투찰 시 낙찰 불가 | % 단위 |
| **A값** | 적격심사용 기준금액 | |
| **공사기간** | 공사 이행기간 | 일 단위 |
| **공고일** | 공고 등록일 | |
| **입찰마감일** | 입찰서 제출 마감일 | |
| **개찰일** | 개찰(낙찰자 발표)일 | |
| **낙찰률** | 낙찰자 투찰률 (파란색 강조) | 개찰 완료 시 표시 |
| **경쟁사 수** | 참여업체 수 | |
| **상태** | 공고중 / 개찰완료 | |

#### 상세 정보 영역 (해당 데이터 있을 경우)

| 필드 | 설명 |
|-----|------|
| **계약방법** | 일반경쟁, 제한경쟁, 지명경쟁 등 |
| **입찰방법** | 전자입찰, 현장입찰 등 |
| **공사현장** | 실제 공사 위치 주소 |
| **참가가능지역** | 입찰 참가 가능 지역 제한 |
| **업종제한** | 참가 가능 업종 자격 조건 |
| **담당자** | 발주기관 담당자명 및 연락처 |

> **주의:** G2B API에서 해당 필드가 제공되지 않는 공고는 표시되지 않습니다.

#### 투찰률 분포 차트

- 개찰 완료 공고에서만 표시
- X축: 참여업체명 (6자 이내 약칭)
- Y축: 투찰률 (%)
- **빨간 막대:** 낙찰 업체
- **파란 막대:** 낙찰 외 업체
- 차트를 통해 낙찰 투찰률 위치와 경쟁사 분포를 한눈에 파악

#### 참여업체 현황 테이블

| 컬럼 | 설명 |
|-----|------|
| **순위** | 투찰률 기준 순위 |
| **업체명** | 참여업체명 (낙찰 시 🏆 아이콘) |
| **투찰금액** | 제출한 투찰금액 (억원) |
| **투찰률** | 투찰금액 ÷ 기초금액 × 100 |
| **낙찰** | 낙찰 여부 (✅) |

#### 유사 입찰 사례

- 현재 공고와 텍스트 유사도가 높은 과거 공고 최대 6건
- **유사도 산출:** TF-IDF 벡터화 → 코사인 유사도 계산 (공고명 기준)
- 클릭 시 해당 공고 상세로 이동

### 5.2 활용 방법

1. **낙찰 패턴 분석:** 투찰률 분포 차트에서 낙찰률이 경쟁사들 사이 어디에 위치하는지 확인합니다.
2. **경쟁사 파악:** 참여업체 테이블에서 이 공고에 참여한 경쟁사와 각각의 투찰률을 확인합니다.
3. **AI 추천 연계:** 상단 **AI 투찰률 추천** 버튼 클릭 시 해당 공고 정보가 자동으로 추천 폼에 채워집니다.
4. **유사 사례 참고:** 유사 공고의 과거 낙찰률을 참고해 투찰 전략을 수립합니다.

## 6. AI 투찰률 추천

**경로:** `/recommend`  
**접근 권한:** analyst, admin

### 6.1 화면 구성

로컬 머신러닝 모델(XGBoost + LightGBM)을 활용해 특정 공고 조건에 최적화된 투찰률 범위를 추천합니다.

> **중요:** 이 시스템은 외부 상용 AI(OpenAI, Claude 등)를 사용하지 않습니다. 모든 추론은 서버 내부 로컬 모델에서 실행됩니다.

#### 입력 폼

| 필드 | 필수 | 설명 |
|-----|------|------|
| **발주기관** | ✅ | 드롭다운에서 선택 (DB에 등록된 기관) |
| **공종** | ✅ | 업종 분류 선택 |
| **지역** | ✅ | 공사 지역 선택 |
| **기초금액 (원)** | ✅ | 숫자로 입력 (예: 500000000 = 5억원) |
| **A값 (원)** | 선택 | 적격심사 기준금액 |
| **공사기간 (일)** | 선택 | 공사 이행일수 |

**공고 연계 방식:** 입찰 상세 페이지에서 'AI 투찰률 추천' 버튼을 누르면 발주기관·공종·지역·기초금액·A값·공사기간이 자동으로 채워집니다.

#### 추천 결과 — 투찰률 범위

| 구분 | 설명 |
|-----|------|
| **보수 하단** | 95% 신뢰구간 하한 (매우 공격적 하단) |
| **추천 하단** | 75% 신뢰구간 하한 (적정 범위 최솟값) |
| **중심값 (권장)** | 모델 예측 중심값 — 가장 균형잡힌 투찰률 |
| **추천 상단** | 75% 신뢰구간 상한 (적정 범위 최댓값) |
| **보수 상단** | 95% 신뢰구간 상한 (매우 안정적 상단) |

시각화 바에서 범위 분포를 직관적으로 확인할 수 있습니다.

#### 낙찰 예상 확률

- **추천 하단으로 투찰 시** 예상 낙찰 확률 (%)
- **중심값으로 투찰 시** 예상 낙찰 확률 (%)
- **추천 상단으로 투찰 시** 예상 낙찰 확률 (%)

> **해석:** 확률이 높을수록 낙찰 가능성이 높지만, 수익성은 낮아집니다. 전략에 따라 최적 투찰률을 선택하세요.

#### 리스크 평가

| 레벨 | 의미 |
|-----|------|
| **HIGH (높음)** | 경쟁이 매우 치열하거나 과거 낙찰 데이터가 불안정한 조건 |
| **MEDIUM (보통)** | 일반적인 경쟁 조건, 표준 전략 적용 권장 |
| **LOW (낮음)** | 경쟁 강도가 낮고 예측 신뢰도가 높은 조건 |

리스크 요인이 목록으로 표시되며, 각 요인이 투찰 결정에 영향을 주는 이유를 설명합니다.

#### 추천 근거 (설명 가능한 AI — SHAP)

모델이 왜 특정 투찰률을 추천했는지 주요 기여 요인을 시각화합니다.

| 요소 | 설명 |
|-----|------|
| **초록 막대** | 투찰률을 높이는 방향으로 기여한 요인 |
| **빨간 막대** | 투찰률을 낮추는 방향으로 기여한 요인 |
| **SHAP 값** | 해당 요인이 최종 예측값에 미친 수치적 영향 (소수점 4자리) |

주요 기여 요인 예시:
- `agency_avg_rate`: 해당 기관의 과거 평균 낙찰률 패턴
- `region_avg_rate`: 해당 지역의 과거 평균 낙찰률
- `amount_log`: 기초금액 규모 (대형 공사일수록 낙찰률이 낮아지는 경향)
- `competition_level`: 과거 유사 공고 평균 경쟁 업체 수

#### 모델 정보 패널

| 항목 | 설명 |
|-----|------|
| **모델 버전** | 학습된 모델 버전 (재학습 시 업데이트) |
| **기준 데이터** | 모델 학습에 사용된 낙찰 결과 건수 |
| **기준 투찰률** | 학습 데이터 전체 낙찰률 평균 |

#### 유사 입찰 사례 테이블

모델이 참조한 유사 과거 공고를 표시합니다. 클릭 시 해당 공고 상세로 이동합니다.

| 컬럼 | 설명 |
|-----|------|
| **공고명** | 유사 과거 공고명 |
| **기관** | 발주기관명 |
| **기초금액** | 공고 기초금액 |
| **낙찰률** | 실제 낙찰 투찰률 |
| **경쟁사** | 참여업체 수 |
| **유사도** | 현재 입력 조건과의 유사도 (%) |

### 6.2 ML 모델 상세 설명

#### 모델 구조

```
입력 특성 (Features)
├── agency_avg_rate       : 기관별 과거 평균 낙찰률
├── agency_bid_count      : 기관별 과거 입찰 건수
├── industry_avg_rate     : 업종별 과거 평균 낙찰률
├── region_avg_rate       : 지역별 과거 평균 낙찰률
├── amount_log            : log10(기초금액) — 금액 규모 정규화
├── competition_level     : 과거 유사 공고 평균 참여업체 수
├── a_value_ratio         : A값/기초금액 비율 (있을 경우)
└── construction_period   : 공사기간 (있을 경우)

출력
├── XGBoost Quantile (q=0.25) → 추천 하단
├── XGBoost Quantile (q=0.50) → 중심값
├── XGBoost Quantile (q=0.75) → 추천 상단
├── XGBoost Quantile (q=0.05) → 보수 하단
└── XGBoost Quantile (q=0.95) → 보수 상단
```

#### 낙찰 확률 계산

LightGBM 이진 분류 모델로 `투찰률 X`에서 낙찰될 확률을 추정합니다.
- 학습 레이블: 해당 투찰률로 낙찰 여부 (1/0)
- 입력: 위 features + 제안 투찰률

#### 모델 재학습

관리자 페이지에서 수동 재학습을 트리거할 수 있습니다.
- 재학습 조건: 개찰 완료 데이터 100건 이상
- 재학습 후 `models/meta.json` 버전 업데이트

#### A값 · 낙찰하한가 자동 계산 카드 (신규)

공고 조건 입력 후 자동으로 A값과 낙찰하한가 범위를 산출합니다.

| 항목 | 설명 |
|-----|------|
| **A값 (예정가격 추정)** | 기초금액 × 사정율 중앙값으로 역산 |
| **낙찰하한가** | 업종별 낙찰하한율 × A값 |
| **P10~P90 범위** | 과거 낙찰 데이터 기반 투찰 범위 분포 |

> **낙찰하한율 기준:** 업종별 고시 하한율 적용 (토목 87.745%, 건축 86.745%, 전기·통신 등 별도)

#### 사정율 트렌드 뱃지 (신규)

추천 결과 상단에 선택한 발주처×공종의 최근 사정율 트렌드를 표시합니다.
- ↑ 빨강: 최근 3개월 사정율 상승 추세
- ↓ 파랑: 하락 추세
- → 회색: 안정적

#### 사정율 신뢰도 뱃지 (신규)

투찰 범위 카드 하단에 사정율 예측 근거와 신뢰도를 표시합니다.

| 뱃지 | 의미 |
|-----|------|
| **inpo21c 실측 N건 기반** (파란색) | inpo21c에서 수집한 N건의 실제 낙찰 데이터로 사정율 예측 |
| **G2B 통계 기반** (회색) | G2B 낙찰 이력 평균으로 사정율 예측 (실측 데이터 없음) |

- **신뢰도 바:** inpo21c 실측 건수가 많을수록 길어지는 가로 진행 표시 (최대 85%)
- inpo21c 실측 건수 3건 이상 시 Bayesian 블렌딩 자동 적용 → G2B 통계 대비 더 정확한 예정가격 예측
- 대전광역시 예시: G2B 기반 0.874 → inpo21c 실측 26건 적용 후 0.9854 (실측값 0.9983, 오차 -1.3%p)

#### 프리즘 2.0 히트맵 (신규)

복수예가 방식 공고의 71개 구간(0.001 단위)별 낙찰확률을 시각화합니다.

| 항목 | 설명 |
|-----|------|
| **BarChart 히트맵** | 구간별 낙찰확률, 높은 구간은 주황색 강조 |
| **TOP10 테이블** | 낙찰확률 상위 10개 구간, 예상 투찰금액 포함 |
| **낙찰하한 필터** | 하한 미만 구간은 자동 제외 |

> **활용:** 프리즘형 예가 공고 투찰 시 TOP10 구간 중 경쟁이 덜 몰릴 것으로 판단되는 구간을 선택합니다.

#### 전략 레포트 PDF 출력 (신규)

추천 결과를 A4 1~2페이지 PDF로 저장합니다.

**레포트 구성:**
1. 공고 기본 정보 (공고명·발주처·기초금액·개찰일)
2. A값·낙찰하한가 요약
3. 사정율 트렌드 (발주처×공종)
4. 4전략 추천요율 + 낙찰확률
5. 프리즘 TOP5 구간 + 리스크 요약

> 우측 상단 **전략 레포트 출력** 버튼 클릭 → PDF 다운로드 (파일명: `전략레포트_공고명_날짜.pdf`)

### 6.3 ML 모델 상세 설명

#### 모델 구조

```
입력 특성 (Features)
├── agency_avg_rate       : 기관별 과거 평균 낙찰률
├── agency_bid_count      : 기관별 과거 입찰 건수
├── industry_avg_rate     : 업종별 과거 평균 낙찰률
├── region_avg_rate       : 지역별 과거 평균 낙찰률
├── amount_log            : log10(기초금액) — 금액 규모 정규화
├── competition_level     : 과거 유사 공고 평균 참여업체 수

```

#### 낙찰 확률 계산

LightGBM 이진 분류 모델 + Monte Carlo 시뮬레이션(50,000회)으로 낙찰확률을 추정합니다.
- **기본:** 학습 레이블 기반 이진 분류
- **실증 분포 적용:** inpo21c 31,800건 이상 수집 시 경쟁사 실제 투찰 분포를 시뮬레이션에 반영

#### 개인 편향 보정

내 입찰 이력이 10건 이상 누적되면 자동으로 개인 투찰 성향 편향을 감지하고 추천값에 보정합니다.
- **보정 방향:** "항상 낙찰자보다 X% 높게 씀" → 추천값 하향 조정 (최대 0.8%)
- **지수 감쇠 가중치:** 최근 투찰일수록 더 높은 가중치 적용

#### 모델 재학습

관리자 페이지에서 수동 재학습을 트리거할 수 있습니다.
- 재학습 조건: 개찰 완료 데이터 100건 이상
- 재학습 후 `models/meta.json` 버전 업데이트

### 6.4 활용 방법

1. **입찰 상세에서 연계:** 입찰 상세 화면에서 'AI 투찰률 추천' 버튼을 누르면 조건이 자동 입력됩니다.
2. **A값 먼저 확인:** A값 카드에서 낙찰하한가를 확인하고, 추천값이 하한 이상인지 검증합니다.
3. **프리즘 히트맵 활용:** 복수예가 공고에서 TOP10 구간 중 경쟁 분산을 고려해 최종 투찰 구간을 결정합니다.
4. **시나리오 비교:** 같은 조건에서 기초금액만 변경해 금액별 추천 투찰률 차이를 비교합니다.
5. **리스크 HIGH 대응:** 리스크가 HIGH인 경우 중심값보다 보수적(상단 방향)으로 투찰하는 전략을 고려합니다.
6. **PDF 레포트 저장:** 입찰 결정 근거를 PDF로 저장해 내부 결재 자료로 활용합니다.

## 7. 내 입찰 관리

**경로:** `/my-bids`  
**접근 권한:** analyst, admin

자사 투찰 이력을 등록·관리하고 AI 추천 정확도 및 낙찰 패턴을 분석합니다.

### 7.1 투찰 이력 탭

#### 이력 등록

| 필드 | 필수 | 설명 |
|-----|------|------|
| **공고 선택** | ✅ | 시스템에 수집된 공고 검색 후 선택 |
| **투찰 요율** | ✅ | 실제 제출한 투찰률 (%) |
| **투찰 금액** | ✅ | 실제 제출한 투찰금액 (원) |
| **투찰일** | ✅ | 투찰 제출일 |
| **메모** | - | 내부 전략 메모 |

> **자동 연계:** 매일 18:00 낙찰 결과 수집 시 내 투찰 이력과 낙찰 결과가 자동으로 연결됩니다 (낙찰/유찰 여부, 낙찰자 요율 자동 기록).

#### 결과 필터

- **전체 / 진행중 / 낙찰 / 유찰** 탭으로 이력을 필터링합니다.
- 낙찰 건은 초록 배지, 유찰 건은 회색 배지로 표시됩니다.

### 7.2 정확도 분석 탭

AI 추천값과 실제 투찰률·낙찰률을 비교해 추천 정확도를 평가합니다.

- **산점도:** X축 = 추천 요율, Y축 = 실제 투찰률 (대각선에 가까울수록 정확)
- **MAE 월별 차트:** 월평균 절대 오차(Mean Absolute Error) 추이

### 7.3 역산 분석 탭 (신규)

낙찰자와 내 투찰률의 격차(rate_diff) 분포를 시각화합니다.

**rate_diff 정의:** `내 투찰률 − 낙찰자 투찰률`
- 양수 (+): 낙찰자보다 높게 씀 (보수적 투찰)
- 음수 (−): 낙찰자보다 낮게 씀 (과공격적 투찰)

#### 히스토그램

- X축: rate_diff 구간 (0.005 단위)
- Y축: 해당 구간 건수
- 빨간 세로선: 중앙 기준선 (0)
- 분포 중심이 오른쪽으로 치우치면 "항상 높게 쓰는" 패턴

#### 핵심 지표

| 지표 | 설명 |
|-----|------|
| **평균 격차** | 전체 이력의 rate_diff 평균 |
| **낙찰 가능 구간** | "X% 낮게 투찰하면 낙찰 가능 구간 진입" 계산 |
| **편향 방향** | too_high(항상 높게) / too_low(낮게) / mixed |

#### 개인 편향 보정 요약

PersonalBiasAnalyzer가 분석한 보정 방향과 신뢰도를 카드로 표시합니다.
보정 신뢰도가 50% 이상이면 AI 추천값에 자동 반영됩니다.

### 7.4 승률 패턴 진단 탭 (신규)

내 투찰 이력 10건 이상 누적 시 자동으로 활성화됩니다. 다양한 차원에서 자사 승률 패턴을 분석합니다.

#### 월별 승률 추이

- X축: 최근 12개월
- Y축: 해당 월 낙찰율 (%) — 낙찰 건수 ÷ 투찰 건수 × 100
- 추세선으로 승률 개선·하락 방향 파악

#### 투찰구간별 낙찰 분포

0.005 단위 구간별로 낙찰 횟수와 유찰 횟수를 스택 막대로 표시합니다.
- 초록: 낙찰 건수
- 회색: 유찰 건수
- **고승률 구간** (파란 뱃지): 낙찰률이 높고 투찰 빈도도 높은 황금 구간

#### 공종별 승률 분포

공종(업종)별 낙찰률을 RadarChart로 시각화합니다. 주력 공종과 취약 공종 파악에 활용합니다.

#### 핵심 진단 지표

| 지표 | 설명 |
|-----|------|
| **최고 승률 구간** | 역대 낙찰이 집중된 투찰률 구간 |
| **평균 승률** | 전체 이력 기준 낙찰률 |
| **최강 공종** | 가장 높은 낙찰률을 기록한 공종 |
| **개선 권고** | 승률이 낮은 구간·공종에 대한 전략 개선 메시지 |

### 7.5 활용 방법

1. **이력 꾸준히 등록:** 10건 이상 누적 시 역산 분석 및 개인 편향 보정이 활성화됩니다.
2. **역산 분석 정기 확인:** 월 1회 역산 분석 탭에서 내 투찰 성향 패턴을 파악합니다.
3. **편향 보정 확인:** "항상 높게 쓴다"는 진단이 나오면 다음 입찰부터 AI 추천값에서 보정분만큼 낮춰 투찰합니다.
4. **승률 패턴 활용:** 승률 패턴 탭에서 황금 구간을 파악하고 해당 구간에 집중 투찰 전략을 수립합니다.

## 8. 경쟁사 분석

**경로:** `/competitors`  
**접근 권한:** 전 역할

### 8.1 화면 구성

입찰 참여 이력이 있는 경쟁사들의 행동 패턴을 분석합니다.

#### 좌측 패널 — 경쟁사 목록

**필터 컨트롤:**

| 컨트롤 | 설명 |
|-------|------|
| **리스크 필터** | 전체 / HIGH / MEDIUM / LOW — 리스크 등급별 필터링 |
| **업체명 검색** | 업체명 키워드 검색 (Enter 또는 검색 버튼) |
| **페이지네이션** | 15건씩 표시, 이전/다음 이동 |

**목록 항목:**

| 필드 | 설명 |
|-----|------|
| **업체명** | 경쟁사 이름 |
| **리스크 등급** | HIGH(빨강) / MEDIUM(노랑) / LOW(초록) 배지 |
| **수주율** | 해당 업체의 낙찰 성공률 (%) |
| **평균 투찰률** | 해당 업체 투찰률 평균 (%) |

#### 우측 패널 — 업체 상세 분석

목록에서 업체 클릭 시 우측에 상세 분석이 표시됩니다.

**핵심 지표 (6개 카드):**

| 지표 | 설명 |
|-----|------|
| **총 입찰** | 이 업체가 참여한 전체 입찰 건수 |
| **수주 건수** | 낙찰 성공 건수 |
| **수주율** | 수주건수 ÷ 총입찰 × 100 (%) |
| **평균 투찰률** | 전체 투찰률 평균 (%) |
| **공격성** | 공격성 점수 (0~10) |
| **일관성** | 투찰률 일관성 점수 (0~10) |

**행동 패턴 레이더 차트:**

5가지 차원으로 경쟁사 특성을 시각화합니다.

| 축 | 산출 기준 |
|--|---------|
| **공격성** | aggression_score × 10 (낙찰하한선 근처 투찰 빈도) |
| **일관성** | consistency_score × 10 (투찰률 표준편차 역수 기반) |
| **낙찰률** | win_rate × 100 (낙찰 성공 빈도) |
| **활동량** | min(total_bids / 2, 100) (입찰 건수 정규화) |
| **안전성** | max(0, 100 - aggression_score × 10) (공격성 역치) |

**자주 함께 참여하는 업체:**

이 경쟁사와 동일 공고에 자주 참여한 업체 Top 7을 공동 참여 횟수와 함께 표시합니다.
→ 해당 업체가 있는 공고에 같이 나타나는 업체 파악에 활용

**월별 활동 추이 차트:**

- **회색 선:** 월별 입찰 참여 건수 (좌축)
- **파란 선:** 월별 평균 투찰률 % (우축)
- 업체의 활동량 증감 패턴과 투찰률 계절성 파악

### 8.2 리스크 등급 산출 기준

리스크 등급은 DB에 저장되지 않고 요청 시마다 실시간으로 계산됩니다.

```
리스크 점수 = aggression_score × 0.4
            + win_rate × 10 × 0.4
            + consistency_score × 0.2

HIGH:   리스크 점수 ≥ 6.0
MEDIUM: 4.0 ≤ 리스크 점수 < 6.0
LOW:    리스크 점수 < 4.0
```

| 요소 | 가중치 | 설명 |
|-----|-------|-----|
| **aggression_score** | 40% | 낙찰하한선에 근접한 공격적 투찰 빈도 (0~10점) |
| **win_rate × 10** | 40% | 낙찰 성공률 (높을수록 더 위협적) |
| **consistency_score** | 20% | 투찰률 일관성 (높을수록 패턴 예측 가능 → 리스크↑) |

#### 투찰구간 탭 (신규)

해당 경쟁사가 최근 투찰한 요율 구간 분포를 실시간으로 확인합니다.

| 항목 | 설명 |
|-----|------|
| **BarChart** | 0.005 단위 버킷별 투찰 빈도, 피크 구간 주황 강조 |
| **기간 토글** | 90일 / 180일 전환 |
| **피크 구간** | 가장 자주 투찰하는 구간 (주황 뱃지) |
| **회피 추천** | 피크 구간 ±0.005 범위를 피하도록 추천 뱃지 표시 |

> **활용:** 입찰 전 주요 경쟁사의 최근 피크 구간을 확인하고, 해당 구간과 다른 구간에 투찰해 직접 경쟁을 회피합니다.

#### 행동 예측 탭 (신규)

선택한 경쟁사가 특정 공고에 참여할 확률과 예상 투찰구간을 ML로 예측합니다.

| 항목 | 설명 |
|-----|------|
| **공고 선택** | 예측할 입찰 공고 번호 검색 |
| **참여 확률** | 0~100% — 해당 경쟁사가 이 공고에 참여할 가능성 |
| **예상 투찰구간** | 과거 투찰 패턴 기반 예상 투찰률 범위 (0.005 단위 BarChart) |
| **TOP5 경쟁사 예측** | 이 공고 참여 가능성이 높은 상위 5개 경쟁사 목록 (Dialog) |

> **활용:** 입찰 전 예상 경쟁사를 파악하고, AI 투찰률 추천의 '알려진 경쟁사' 항목에 반영해 Monte Carlo 시뮬레이션 정확도를 높입니다.

### 8.3 활용 방법

1. **위협 경쟁사 선별:** 리스크 필터를 'HIGH'로 설정해 가장 위협적인 경쟁사를 우선 분석합니다.
2. **투찰 전략 수립:** 주요 경쟁사의 평균 투찰률과 공격성을 파악해 투찰률 포지셔닝을 결정합니다.
3. **투찰구간 회피:** 경쟁사가 집중하는 피크 구간을 확인하고 의도적으로 다른 구간에 투찰합니다.
4. **공동 참여 네트워크:** '자주 함께 참여하는 업체'를 통해 특정 기관 공고에 묶음으로 참여하는 업체 그룹을 파악합니다.
5. **활동 패턴 분석:** 월별 추이 차트에서 특정 분기에 활동이 급증하는 업체는 계절성 전략을 사용할 가능성이 있습니다.

## 9. 예가 분석

복수예가 방식 공고에서 예가 번호 조합의 당첨 빈도 패턴을 분석합니다.

15개 번호 조합(C(15,4) = 1,365가지)의 당첨 빈도를 BarChart로 표시합니다.
- X축: 번호 조합 인덱스 (1~1365)

#### 발주처 특화 패턴 (신규)

발주처 드롭다운에서 기관을 선택하면 해당 발주처의 과거 낙찰 데이터에서 역산한 번호별 빈도를 별도 표시합니다.

| **번호 1~15 빈도** | 해당 발주처의 예가 번호별 등장 빈도 |
| **TOP3 하이라이트** | 가장 자주 등장한 번호 3개 주황 강조 |
| **dominant_zone 뱃지** | 특정 번호 대역(예: 상위/중위/하위)이 집중되는 패턴 |

### 9.2 발주처 예가 위치 패턴 탭 (신규)

**경로:** 발주기관 목록 → 기관명 클릭 → AgencyDetailPage → **예가패턴** 탭

inpo21c 실측 데이터에서 집계한 위치별 추첨 가중치를 시각화합니다.

| 항목 | 설명 |
|-----|------|
| **15-bar 차트** | 위치 1~15의 실측 추첨 확률 (파란 막대, 균등=6.7% 점선 기준선) |
| **상위 3개 위치** | 파란 강조 + 퍼센트 레이블 (가장 자주 추첨된 위치) |

| **샘플 수** | 분석에 사용된 inpo21c 실측 건수 |

> **활용:** 특정 발주처에 자주 입찰하는 경우, 해당 기관의 예가 위치 편중 패턴을 파악해 복수예가 추첨 시뮬레이션 전략에 반영합니다. 이 가중치는 AI 투찰률 추천의 Monte Carlo 시뮬레이션에도 자동으로 적용됩니다.

### 9.4 활용 방법

1. **발주처 선택:** 자주 입찰하는 발주처를 선택해 해당 기관의 번호 편중 패턴을 파악합니다.
2. **TOP3 번호 참고:** 빈도가 높은 번호 대역을 참고하되, 매번 동일 조합을 선택하면 경쟁자도 예측할 수 있음을 주의합니다.
3. **발주처 예가패턴 탭 연계:** AgencyDetailPage 예가패턴 탭에서 위치 가중치를 확인하고 프리즘 2.0 히트맵과 함께 활용합니다.

## 10. 공동도급

### 10.1 협정사 탐색

경쟁사 DB(25,387개)에서 공동도급 협정 가능 업체를 필터링합니다.

공고를 선택한 후 **적격심사 AI 매칭** 버튼을 클릭하면, 해당 공고의 기초금액·업종 기준으로 적격심사 통과 가능한 협정사를 자동으로 제안합니다.

### 10.3 적격심사 시뮬레이터 (신규)

입찰 상세 페이지 → **공동도급 시뮬레이터** 버튼으로 진입합니다. 파트너 구성과 지분율을 실시간으로 조정하며 적격심사 통과 여부를 사전 시뮬레이션합니다.

| 기초금액 구간 | 통과 threshold |
|------------|--------------|
| ~ 10억 | 12점 이상 |
| 10억 ~ 50억 | 14점 이상 |
| 50억 ~ 100억 | 16점 이상 |
| 100억 초과 | 18점 이상 |

### 10.4 활용 방법

1. **혼자 따기 어려운 공고 확인:** 기초금액이 크거나 실적 요건이 높은 공고에서 AI 매칭을 실행합니다.

3. **지분율 협의:** 제안된 최소 지분율을 협정서 작성 시 기준점으로 활용합니다.
4. **시뮬레이터 사전 검증:** 파트너 확정 전 시뮬레이터에서 다양한 지분율 조합을 테스트해 통과 가능한 최적 구성을 도출합니다.

## 12. 통계 분석

**경로:** `/statistics`  
**접근 권한:** 전 역할

### 12.1 공통 컨트롤

페이지 상단의 **최근 N개월** 드롭다운으로 분석 기간을 설정합니다.

| 옵션 | 기간 |
|-----|------|
| 3개월 | 최근 90일 |
| 6개월 | 최근 180일 |
| 12개월 | 최근 365일 (기본값) |
| 24개월 | 최근 730일 |

> 기간을 변경하면 **모든 탭의 데이터**가 즉시 업데이트됩니다 (히트맵 제외).

### 12.2 탭 1 — 개요

전체 시장의 거시적 동향을 파악합니다.

#### 핵심 지표 카드 (4개)

| 지표 | 산출 기준 |
|-----|---------|
| **총 입찰 건수** | 선택 기간 내 전체 공고 수 |
| **낙찰 건수** | 개찰 완료된 공고 수 |
| **평균 낙찰률** | 낙찰자 투찰률 평균 × 100 |
| **평균 경쟁강도** | 공고당 평균 참여업체 수 |

#### 월별 추이 차트

시간에 따른 입찰 건수와 낙찰률 변화를 이중 축으로 표시합니다.

#### 투찰률 분포 히스토그램

낙찰 투찰률이 어떤 범위에 몰려 있는지 분포도를 보여줍니다.
- X축: 투찰률 구간 (0.1% 단위)
- Y축: 해당 구간의 공고 건수
- 분포의 피크가 최적 투찰률 대역을 시사합니다.

#### 발주기관별 현황 테이블

| 컬럼 | 설명 |
|-----|------|
| 기관명 | 발주기관명 |
| 입찰 건수 | 선택 기간 내 총 공고 수 |
| 평균 낙찰률 | 해당 기관 개찰 완료 공고의 낙찰률 평균 |
| 평균 경쟁강도 | 해당 기관 공고 평균 참여업체 수 |

#### 공종×월 낙찰률 히트맵

- 행: 월 (1~12월)
- 열: 주요 공종 (상위 6개)
- 셀 색상: 낙찰률 높을수록 짙은 파란색
- **데이터 기준:** 최근 24개월 고정 (월 기간 선택 미적용)

### 12.3 탭 2 — Level 1: 기관/지역/공종

#### 지역별 분석

| 컬럼 | 설명 |
|-----|------|
| 지역명 | 광역시·도 단위 |
| 입찰 건수 | 선택 기간 내 건수 |
| 평균 낙찰률 | 해당 지역 공고 낙찰률 평균 |
| 평균 경쟁강도 | 해당 지역 평균 참여업체 수 |
| 낙찰률 범위 | 최솟값 ~ 최댓값 |

지역별 낙찰률 막대 차트가 함께 표시됩니다.

#### 공종별 분석

지역별 분석과 동일한 구조로 공종(업종) 단위로 비교합니다.

### 12.4 탭 3 — Level 2: 클러스터 분석

K-Means 알고리즘으로 유사한 입찰 공고를 그룹화해 시장 세분화를 보여줍니다.

#### 클러스터링 기준

| 특성 | 설명 |
|-----|------|
| **금액 규모** | log10(기초금액) — 대형/중형/소형 분류 |
| **평균 낙찰률** | 해당 공고의 낙찰 투찰률 |
| **경쟁 업체 수** | 참여업체 수 |

- 기본 클러스터 수: 4개 (K=4)
- 최대 분석 데이터: 최근 500건

#### 클러스터 카드 (각 그룹별)

| 항목 | 설명 |
|-----|------|
| 클러스터 번호 | 색상으로 구분 |
| 건수 | 해당 그룹 공고 수 |
| 평균 금액 | 그룹 내 평균 기초금액 |
| 평균 낙찰률 | 그룹 내 평균 낙찰률 |
| 평균 경쟁업체 | 그룹 내 평균 참여업체 수 |
| 주요 공종 | 해당 그룹의 가장 빈번한 공종 |

#### 산점도

- X축: 기초금액 (억원)
- Y축: 낙찰률 (%)
- 색상: 클러스터 번호별 구분

### 12.5 탭 4 — Level 3: ML 모델

ML 파이프라인의 현황과 데이터 준비도를 모니터링합니다.

#### ML 파이프라인 상태

#### 데이터 준비도

| 항목 | 기준 |
|-----|------|
| 총 낙찰 데이터 | 학습에 사용된 개찰 결과 수 |

### 12.6 활용 방법

1. **투찰률 적정 대역 파악:** 투찰률 분포 히스토그램에서 낙찰이 집중된 구간을 찾아 기본 전략 범위를 설정합니다.

## 15. 데이터 제공 기준 및 산출 근거

### 15.1 데이터 출처

| 데이터 | 출처 | 수집 방식 |
|-------|-----|---------|
| 입찰 공고 | 조달청 나라장터 G2B OpenAPI | `getBidPblancListInfoCnstwk` |
| 낙찰 결과 | 조달청 나라장터 G2B OpenAPI | `getBidPblancRsltListInfoCnstwkCo` |
| 전 참여자 데이터 | inpo21c (cloud.info21c.net) | 세션 쿠키 기반 스크래핑 (매주 월) |

### 15.2 주요 지표 산출 공식

| 지표 | 공식 |
|-----|------|
| **투찰률** | `투찰금액 ÷ 기초금액 × 100` (%) |
| **낙찰률** | 낙찰된 투찰률 (입찰 결과의 낙찰자 투찰률) |
| **수주율** | `낙찰 건수 ÷ 총 입찰 참여 건수 × 100` (%) |
| **공격성 점수** | `낙찰하한선 ± 0.5% 이내 투찰 건수 ÷ 총 투찰 건수 × 10` |
| **일관성 점수** | `10 - min(10, 투찰률 표준편차 × 1000)` |
| **리스크 점수** | `aggression × 0.4 + win_rate × 10 × 0.4 + consistency × 0.2` |
| **유사도** | TF-IDF 벡터 코사인 유사도 (공고명 기준) |
| **rate_diff** | `내 투찰률 − 낙찰자 투찰률` (역산 분석 기준) |
| **궁합 점수** | 실적·지역·업종 기반 공동도급 협정 적합도 (0~100) |

### 15.3 데이터 한계 및 주의사항

1. **수집 지연:** G2B API 제공 시각에 따라 당일 공고가 익일 반영될 수 있습니다.
2. **낙찰 결과 미확정:** 개찰일이 지났으나 결과가 아직 API에 미반영된 경우 상태가 '공고중'으로 유지될 수 있습니다.
3. **신규 필드 반영:** 2026년 5월 이전 수집된 공고는 `공사현장`, `계약방법` 등 상세 필드가 비어있을 수 있습니다.
4. **ML 예측 신뢰도:** 학습 데이터가 100건 미만이거나 특정 기관·지역 데이터가 부족한 경우 예측 정확도가 낮아질 수 있습니다.
5. **경쟁사 범위:** 시스템에 등록된 경쟁사는 G2B 수집 데이터에 입찰 결과가 존재하는 업체에 한합니다.
6. **inpo21c 의존성:** inpo21c 쿠키가 만료되면 전 참여자 데이터 수집이 중단됩니다. 관리자 페이지에서 쿠키 상태를 주기적으로 확인하세요.
7. **예가 번호 역산:** 통계적 추정이며 실제 당첨 번호를 보장하지 않습니다.

## 16. 자주 묻는 질문 (FAQ)

**Q1. 최신 공고가 입찰현황에 보이지 않습니다.**  
A. G2B API는 공고 등록 후 최대 수 시간의 배포 지연이 있습니다. 당일 09:00, 15:00 자동 수집 이후 반영됩니다. 즉시 확인이 필요한 경우 관리자에게 수동 수집을 요청하세요.

**Q2. 투찰률 분포 차트가 일부 공고에서 표시되지 않습니다.**  
A. 해당 공고의 낙찰 결과(bid_results)가 아직 수집되지 않은 상태입니다. 개찰 완료 후 15:00 수집 이후에 표시됩니다.

**Q3. AI 투찰률 추천 결과가 과거 실제 낙찰률과 많이 다릅니다.**  
A. 특정 기관·지역·공종 조합의 학습 데이터가 부족한 경우 발생할 수 있습니다. 유사 입찰 사례 섹션에서 직접 참고하고, SHAP 기여 요인으로 차이 원인을 확인하세요. 데이터가 충분히 쌓인 후 모델 재학습을 권장합니다.

**Q4. 경쟁사 리스크가 항상 MEDIUM으로 표시됩니다.**  
A. 해당 경쟁사의 입찰 참여 건수가 적어 공격성·일관성 점수 계산이 정확하지 않을 수 있습니다. 일반적으로 10건 이상 참여 이력이 있을 때 신뢰도 있는 리스크 등급이 산출됩니다.

**Q5. 통계의 N개월 설정을 변경해도 일부 탭이 갱신되지 않는 것 같습니다.**  
A. 탭 간 이동 시 해당 탭 데이터가 로드됩니다. 탭을 클릭한 후 잠시 기다리면 선택한 기간으로 데이터가 갱신됩니다. (히트맵은 24개월 고정 기준으로 기간 설정이 적용되지 않습니다.)

**Q6. 비밀번호를 잊었습니다.**  
A. 관리자 페이지의 사용자 관리에서 관리자가 비밀번호를 재설정할 수 있습니다. 관리자에게 문의하세요.

**Q7. 모델 재학습은 언제 해야 하나요?**  
A. 신규 낙찰 결과가 100건 이상 누적된 시점, 또는 AI 추천 결과가 실제 낙찰과 지속적으로 차이가 발생할 때 재학습을 권장합니다. 통상 월 1~2회 재학습으로 모델 정확도를 유지할 수 있습니다.

**Q8. A값 자동 계산 결과가 실제와 다릅니다.**  
A. A값은 과거 사정율 중앙값 기반 추정치입니다. 실제 A값은 복수예비가격 추첨 결과에 따라 달라지므로 참고용으로만 활용하고, 공고에 A값이 명시된 경우 해당 값을 우선합니다.

**Q9. 프리즘 2.0 히트맵에서 TOP10 구간이 항상 비슷합니다.**  
A. 동일 발주처·공종 조합의 과거 데이터가 집중되어 있으면 특정 구간 편중이 나타날 수 있습니다. 경쟁사 투찰구간 탭과 함께 확인하여 경쟁이 몰리는 구간을 피하는 전략을 사용하세요.

**Q10. 역산 분석 탭에 데이터가 없습니다.**  
A. 내 입찰 이력이 낙찰 결과와 연계된 건이 최소 3건 이상 필요합니다. 이력을 등록하고 낙찰 결과가 자동 연계될 때까지 기다리거나, 수동으로 결과를 업데이트하세요.

**Q11. inpo21c 쿠키를 어떻게 갱신하나요?**  
A. 관리자 페이지 → 수집 현황 탭 → inpo21c 섹션에서 쿠키 갱신 버튼을 클릭하거나, 서버 환경변수 `INPO21C_COOKIE`를 직접 갱신합니다. 쿠키는 cloud.info21c.net에 로그인 후 브라우저 개발자 도구에서 복사합니다.

**Q12. 공동도급 AI 매칭에서 모든 업체가 부적격으로 나옵니다.**  
A. 해당 공고의 기초금액 기준 적격심사 요건이 높거나, inpo21c 데이터에 실적 정보가 부족한 경우 발생할 수 있습니다. 경쟁사 DB에 실적 데이터가 없는 업체는 적격 여부 판단이 보수적으로 처리됩니다.
